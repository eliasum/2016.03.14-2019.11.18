I. Используя Updater(Using The Updater):
----------------------------------------------------------------------------------------
I.I. Создать оригинальный установочный пакет:
1. Основная страница адгоритма: http://www.advancedinstaller.com/user-guide/tutorial-updater.html#create
2. Создаем проект простой установки по порядку до момента постройки проекта, т.е. до пункта 3: http://www.advancedinstaller.com/user-guide/tutorial-simple.html
3. Добавить 'Программу обновления' в левой панели проекта.
4. Выставить флаг 'Проверять наличие обновлений с помощью Advanced Installer'.
5. Ввести 'URL-адрес файла конфигурации обновлений' http://api.nccp-eng.ru/test/StoryUpdates.txt.
Сам файл конфигурации обновлений StoryUpdates.txt будет создан автоматически после создания обновления.
6. Создать ярлык Advanced Updater на рабочем столе для запуска обновления.
7. Сохранить, построить и установить пакет, на рабочем столе появится ярлык updater.

I.II. Создать первое обновление:
1. Создать резервную копию оригинального проекта путем копирования.
2. Изменить версию продукта, например, на 2.0.0.
3. При выборе другой страницы вам будет предложен метод обновления. Ответ 'Создать новый' код продукта для того, чтобы включить функцию автоматического обновления.
4. Убедитьcя, что проект содержит значение реестра "Version" на странице реестра, установленное в значение [ProductVersion].
5. Задать имя сборки story2.
6. Сохранить и построить пакет.
7. Создать новый проект конфигурации обновлений StoryUpdates.aip: http://www.advancedinstaller.com/user-guide/tutorial-update.html.
8. Нажать на кнопку 'Обновление' и выбрать файл story2.msi.
9. Изменить 'Основной URL' на http://api.nccp-eng.ru/test/story2.msi.
10. Выбрать обновление из левой панели и переименовать его в "Story 2.0".
11. Загрузить story2.msi пакет на сервер.
12. Построить проект конфигурации обновлений.
13. Загрузить файл StoryUpdates.txt на сервер.
14. Запустить updater на рабочем столе и проверить наличие обновления story2.

I.III. Создать второе обновление:
1. Выполнить всё по такому же алгоритму, как в пункте I.II.

I.IV. Создать третье обновление:
1. Выполнить всё по такому же алгоритму, как в пункте I.II.
