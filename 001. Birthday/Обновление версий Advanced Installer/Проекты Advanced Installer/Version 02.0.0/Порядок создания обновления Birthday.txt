I. Используя Updater(Using The Updater):
----------------------------------------------------------------------------------------
I.I. Создать оригинальный установочный пакет:
1. Основная страница адгоритма: http://www.advancedinstaller.com/user-guide/tutorial-updater.html#create
2. Создаем проект простой установки по порядку до момента постройки проекта, т.е. до пункта 3: http://www.advancedinstaller.com/user-guide/tutorial-simple.html
ПРОГРАММА ОБНОВЛЕНИЯ
3. Добавить 'Программу обновления' в левой панели проекта.
4. Выставить флаг 'Проверять наличие обновлений с помощью Advanced Installer'.
5. Ввести 'URL-адрес файла конфигурации обновлений' http://api.nccp-eng.ru/app/BirthdayUpdates.txt
Сам файл конфигурации обновлений BirthdayUpdates.txt будет создан автоматически после создания обновления.
6. Имя файла обновления - BirthdayUpdater.
7. Выполнение обновления выбрать автоматическое и настроить задание.
8. Во вкладке задание вытавить:
- имя задания - BirthdayUpdater;
- условие установки - %USERNAME = 1;
9. В атрибутах выставить галочку 'Запускать щадание только в том случае, если имеется подключение к Интернету'.
10. Во вкладке расписание выставить задание 'При запуске системы'.
ПАРАМЕТРЫ УСТАНОВКИ
11. В панели 'Определение пакета', 'Параметры установки' выбрать 'Тип Установки' - Только на пользователя.
12. Убрать галочку 'Запуск от имени администратора' и выбрать 'Уровень выполнения' As Invoker.
РЕЕСТР
13. В панели 'Ресурсы', 'Реестр' папку Software\[Manufacturer]\[ProductName] оставить без изменения, но
выполнить для этой папки 'Очистку после удаления' -> Удалить раздел реестра со всеми его подразделами и значениями.
14. Создать папку Software\Microsoft\Windows\CurrentVersion\Run. 
15. В ней создать параметр Birthday с данными [APPDIR]Birthday.exe и BirthdayUpdater с данными 
[APPDIR]BirthdayUpdater.exe.
16. Для каждого из этих параметров выполнить 'Очистку после удаления'.
СБОРКИ
17. В панели 'Определение пакета', 'Сборки' выбрать 'Тип пакета' - Единый MSI-файл, 'имя MSI' - 'Birthday2.0.0' 
и выставит галочку 'Включать CAB-файл в базу данных MSI'.

I.II. Создать первое обновление:
1. Создать резервную копию оригинального проекта путем копирования.
2. Изменить версию продукта, например, на 2.0.1.
3. При выборе другой страницы вам будет предложен метод обновления. Ответ 'Создать новый' код продукта для того, чтобы включить функцию автоматического обновления.
4. Убедитьcя, что проект содержит значение реестра "Version" на странице реестра, установленное в значение [ProductVersion].
5. Задать имя сборки Birthday2.0.1.
6. Заменить в 'Файлах и папках' файл Birthday.exe соответствующим новой версии.
7. Сохранить и построить пакет.

I.III. Создать новый проект конфигурации обновлений:
1. Задать имя BirthdayUpdates.aip: http://www.advancedinstaller.com/user-guide/tutorial-update.html.
2. Нажать на кнопку 'Обновление' и выбрать файл Birthday2.0.0.msi.
3. Изменить 'Основной URL' на http://api.nccp-eng.ru/app/Birthday2.0.0.msi.
4. Выбрать обновление из левой панели и переименовать его в "Birthday2.0.0".
5. Загрузить Birthday2.0.0 пакет на сервер.
6. Построить проект конфигурации обновлений.
7. Загрузить файл BirthdayUpdates.txt на сервер.
8. Запустить updater на рабочем столе и проверить наличие обновления Birthday2.0.0.

I.IV. Создать второе обновление:
1. Выполнить всё по такому же алгоритму, как в пункте I.III.
